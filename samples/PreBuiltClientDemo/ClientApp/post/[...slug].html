This is the /post/[...slug] page

NOTE: This demo doesn't run any Javscript, so you really should see hardcoded placeholders like [...slug].
If this were an actual Next.js app, you could use router.query to render custom content as appropriate.

It should be returned for requests issued to "/post/{*slug}". For example:

  • "/post/123/a"
  • "/post/a/b/c"

It should NOT be returned for requests such as "/post/123", as that should instead serve page "/post/[pid].html".
